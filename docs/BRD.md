## 1. 프로젝트 개요

### 1.1 서비스 소개

**ModuMark** (Modular + Markdown / 모두의 마크다운)는 마크다운(Markdown) WYSIWYG 편집과 PDF 편집을 하나의 애플리케이션에서 제공하는 무료 통합 편집 도구이다. Windows 데스크탑 애플리케이션과 웹(Web) 서비스를 동시에 제공한다.

### 1.2 배경 및 문제 정의

#### 시장 배경

2021년 마크다운 에디터 시장의 표준으로 자리잡았던 **Typora**가 유료 전환($14.99 영구 라이선스)을 단행하면서, 무료 대안을 찾는 사용자 수요가 급증하였다. 이후 Mark Text, Obsidian 등 대안 도구들이 등장했으나 각각 다른 한계를 가지고 있다.

동시에, 개발자·학생·직장인 등 지식 노동자들은 마크다운 문서 작성과 PDF 처리를 별도의 도구로 분리하여 처리해야 하는 비효율적인 워크플로우를 강요받고 있다.

#### 핵심 비즈니스 문제

| # | 문제 | 영향 |
|---|------|------|
| P1 | 마크다운 에디터와 PDF 편집 도구가 분리되어 있어, 사용자가 여러 앱을 오가며 작업해야 함 | 생산성 저하, 컨텍스트 전환 비용 발생 |
| P2 | 품질 있는 마크다운 WYSIWYG 에디터가 모두 유료이거나 기능이 과도하게 복잡함 | 진입 장벽 높음, 학습 비용 발생 |
| P3 | PDF 편집·OCR·변환 도구는 마크다운과 연동되지 않아 문서 파이프라인이 단절됨 | 반복 작업 증가, 데이터 재입력 필요 |
| P4 | 데스크탑 전용 도구는 설치 없이 사용하려는 사용자를 수용하지 못함 | 접근성 제한 |

### 1.3 솔루션 요약

ModuMark는 다음을 하나의 제품으로 통합한다.

- **마크다운 WYSIWYG 편집기**: 실시간 렌더링, 직관적인 편집 경험
- **PDF 편집 도구**: 뷰어, 병합·분할, OCR(광학 문자 인식), 마크다운 변환
- **크로스 플랫폼 접근**: Windows 네이티브 앱 + 웹 브라우저

## 4. 현황 분석 (As-Is)

### 4.1 경쟁사 분석 및 Pain Point

| 경쟁 도구 | 유형 | 강점 | Pain Point (사용자 불만) | ModuMark 차별점 |
|-----------|------|------|--------------------------|----------------|
| **Typora** | 마크다운 에디터 | 최고 수준의 WYSIWYG UX, 깔끔한 UI | 2021년부터 유료화($14.99), 데스크탑 전용, PDF 편집 미지원 | 무료 제공, 웹 지원, PDF 통합 |
| **Mark Text** | 마크다운 에디터 | 무료 오픈소스, WYSIWYG 지원 | 2022년 이후 개발 둔화, 버그 미수정, 유지보수 불투명 | 지속적 업데이트, 안정적 운영 |
| **Obsidian** | 지식 관리 도구 | 강력한 플러그인 생태계, 그래프 뷰 | 과도한 기능으로 학습 비용 높음, 단순 편집 목적에 불필요하게 복잡함 | 단순·직관적 UX, 빠른 시작 |
| **알PDF** | PDF 편집 도구 | 국내 인지도 높음, 다양한 PDF 기능 | 마크다운 미지원, 핵심 기능 일부 유료, 마크다운 연동 불가 | 마크다운과 PDF 완전 통합 |
| **Notion** | 협업 문서 도구 | 블록 에디터, 협업, 클라우드 | 오프라인 불가, 프라이버시 우려, 로컬 파일(.md) 편집 불가 | 로컬 파일 기반, 오프라인 사용 가능 |
| **iLovePDF / Smallpdf** | 온라인 PDF 도구 | 편리한 웹 기반 접근, 다양한 PDF 기능 | **보안 리스크**: 파일이 외부 서버에 업로드·저장됨 → 기밀 문서 보안 사고 위험, 기업 보안 정책 위반 소지 | **로컬 처리**: 파일이 서버에 전송되지 않아 기업 보안 정책 완전 준수 |

### 4.2 시장 기회 (Market Gap)

현재 시장에서 **마크다운 WYSIWYG 편집 + PDF 편집(병합·분할·OCR·변환)을 무료로 통합 제공하는 도구가 존재하지 않는다**. 이것이 ModuMark가 진입할 수 있는 핵심 시장 공백이다.

```
[현재 시장 구도]
마크다운 에디터 진영 ←──────────── 시장 공백 ────────────→ PDF 편집 진영
(Typora, Obsidian, Mark Text)                               (알PDF, Adobe Acrobat)
                              ↑
                         ModuMark
                    (두 영역을 무료로 통합)
```

### 4.3 사용자 현재 워크플로우 (기존 불편함)

```
[현재 사용자 워크플로우]
1. 텍스트 에디터로 .md 파일 작성 (VSCode / Typora 유료 등)
   ↓
2. 마크다운 → PDF 변환 (온라인 도구 별도 사용)
   ↓
3. PDF 편집이 필요한 경우 별도 PDF 도구 실행 (알PDF / Adobe 등)
   ↓
4. OCR이 필요한 경우 또 다른 도구 사용

→ 최소 3~4개 도구, 잦은 컨텍스트 전환, 일부 유료 비용 발생
```

## 5. 목표 상태 (To-Be)

### 5.1 이상적 사용자 경험

```
[ModuMark 목표 워크플로우]
.md 파일 더블클릭 → ModuMark 자동 실행 → 즉시 WYSIWYG 편집
                                              ↓
                             PDF 변환 / PDF 병합 / PDF 분할
                             PDF OCR / PDF 뷰어
                              ↓
                         작업 완료 (단일 앱)
```

### 5.2 핵심 가치 제안 (Value Proposition)

| 가치 | 설명 |
|------|------|
| **통합 (Integration)** | 마크다운 편집과 PDF 처리를 하나의 앱에서 완결 |
| **무료 (Free)** | 핵심 기능 전체 무료 제공, 숨겨진 비용 없음 |
| **접근성 (Accessibility)** | 설치 없이 웹에서 즉시 사용 가능, Windows 앱도 제공 |
| **단순함 (Simplicity)** | 마크다운 문법을 몰라도 WYSIWYG로 직관적 편집 |
| **로컬 우선 (Local-First)** | 파일이 서버에 저장되지 않음, 프라이버시 보호 |
| **보안 (Security)** | 모든 파일 처리가 로컬(클라이언트)에서 수행됨. 파일이 외부 서버에 전송·저장되지 않아 기업 보안 정책 준수 가능. iLovePDF, Smallpdf 등 온라인 도구 대비 기업/개인 문서 보안 보장 |

### 5.3 목표 사용자 경험 시나리오

- **.md 파일 연결**: Windows에서 .md 파일을 더블클릭하면 ModuMark가 자동으로 실행되어 즉시 WYSIWYG 편집 모드로 진입한다.
- **탭 기반 다중 문서**: 새 문서 생성 또는 파일 열기 시 새 창이 뜨지 않고 동일 앱 내 새 탭으로 열린다. 여러 문서를 탭으로 전환하며 동시에 작업할 수 있다.
- **PDF 원클릭 변환**: 마크다운 문서를 작성 완료 후 버튼 한 번으로 PDF로 변환한다.
- **PDF 작업 통합**: PDF 파일 병합, 특정 페이지 분리, OCR로 텍스트 추출, 마크다운으로 변환이 동일 앱 내에서 가능하다.
- **웹 동일 경험**: 웹 브라우저에서도 앱과 동일한 기능을 설치 없이 사용할 수 있다.

### 6.1 포함 범위 (In-Scope)

| **배포** | Windows 설치 파일(.msi/.exe) 코드 서명(Code Signing) 적용 |
| **현지화** | 한국어 UI 기본 제공 |
| **인프라** | 서버리스 백엔드 (OCR 처리), 정적 웹 배포 |
| **SEO 최적화** | SSR/SSG 기반 공개 페이지, 메타태그/OG 태그, sitemap.xml, robots.txt, 구조화 데이터(Schema.org), Core Web Vitals 준수 |

### 6.2 제외 범위 (Out-of-Scope)

| 카테고리 | 제외 이유 |
|----------|----------|
| macOS / Linux 앱 | 1차 출시 범위 초과, 향후 검토 |
| 클라우드 동기화 | 백엔드 복잡도 증가, 개인정보 처리 부담 |
| 실시간 협업 (Co-editing) | 서버 비용 및 개발 복잡도 과다 |

| Mermaid 다이어그램 | 초기 MVP 범위 최소화 |
| 다국어 UI (영어 등) | 한국 시장 집중 후 글로벌 확장 시 추가 |
| 모바일 앱 (iOS / Android) | 플랫폼 개발 리소스 부족, 향후 검토 |

## 8. 제약 조건 및 가정

### 8.1 제약 조건 (Constraints)

| 유형 | 제약 | 영향 |
|------|------|------|
| **조직** | 1인 개발 체제 | 개발 속도 제한, 동시 작업 불가, MVP 범위 최소화 필요 |
| **예산** | 최소 비용 운영 원칙 | 유료 서비스·도구 사용 제한, 오픈소스 우선 |
| **인프라** | Vercel 무료 티어 사용 | 서버리스 함수 실행 시간 제한(10초), 월 대역폭 제한, 대용량 파일 처리 제약 |
| **배포** | Windows 코드 서명 인증서 비용 | EV(Extended Validation) 인증서 비용 연간 약 $200~$400 발생, 미서명 시 SmartScreen 경고 발생 |
| **법적** | 개인정보보호법 준수 | 사용자 파일을 서버에 저장하지 않는 로컬 우선 아키텍처 필수 |

### 8.2 가정 (Assumptions)

| # | 가정 | 검증 방법 |
|---|------|----------|
| A1 | Tauri 2.0이 Next.js 15와 안정적으로 통합 가능하다 | PoC(Proof of Concept) 선행 개발로 검증 |
| A2 | Milkdown 에디터가 Typora 수준의 WYSIWYG 품질을 제공할 수 있다 | 초기 프로토타입 사용성 테스트로 검증 |
| A3 | Tesseract.js의 한국어 OCR 정확도가 실사용 가능 수준(80% 이상)이다 | 한국어 샘플 문서 OCR 테스트로 검증 |
| A4 | Vercel 무료 티어로 초기 트래픽(MAU 10,000명 수준)을 처리할 수 있다 | 부하 테스트 및 Vercel 사용량 모니터링 |
| A5 | Google AdSense 승인이 서비스 출시 후 3개월 이내에 완료된다 | 출시 초기부터 AdSense 신청 진행 |
| A6 | 한국 사용자들이 무료 마크다운+PDF 통합 도구에 충분한 수요를 가지고 있다 | 베타 출시 후 사용자 데이터로 검증 |
