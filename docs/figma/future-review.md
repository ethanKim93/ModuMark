## REVIEW-001: OCR LaTeX 내보내기 (PROPOSAL-004 이관)

| 항목 | 내용 |
|------|------|
| **원 제안** | PROPOSAL-004 (feature-proposals.md) |
| **Figma 위치** | OCR 화면 기능 소개 (`3:542`) |
| **재검토 시점** | Phase 3 완료 후 (또는 학술 사용자 수요 확인 시) |
| **보류 결정일** | 2026-03-07 |

### 기능 설명

OCR 결과를 LaTeX 형식으로 내보내기. 수식·학술 논문 작성 사용자(이수진 페르소나)에게 유용.

### 보류 이유

- 구현 난이도 높음: LaTeX 문법 변환 로직 복잡, 수식 인식 정확도 미지수
- 틈새 시장 (학술 사용자 한정)
- Phase 1·2 핵심 기능 우선 완성 필요

### 재검토 조건

다음 조건 중 하나 이상 충족 시 재검토 진행:
1. Phase 3 완료 후 개발 리소스 여유 확인
2. 학술 사용자 피드백/수요 데이터 확보 (예: GitHub Issues, 사용자 설문)
3. Tesseract.js 또는 대안 OCR 엔진의 수식 인식 정확도 개선 확인

### 구현 고려사항 (재검토 시 참고)

- **LaTeX 변환 접근법**: OCR 텍스트 → 수식 패턴 감지 → LaTeX 마크업 변환
- **대안 검토**: MathpixOCR API (수식 특화 OCR) - 단, 외부 서버 전송 필요 → 로컬 우선 원칙과 충돌 검토 필요
- **범위 제한안**: 간단한 수학 기호만 LaTeX 변환 (완전한 수식 인식 없이)
- **PRD 배치 예상**: Should Have 또는 Could Have (Phase 3)
