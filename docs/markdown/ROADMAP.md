## Phase 1: MVP 핵심 편집기 ✅ 완료 (2026-03-07)

**목표**: Typora 수준의 무료 WYSIWYG 마크다운 편집 + 탭 기반 다중 문서를 사용 가능한 수준으로 제공한다.

### 기능 범위

| 기능 | 우선순위 | 설명 |
|------|---------|------|
| WYSIWYG 편집 (Milkdown) | P0 | 헤딩, 굵게, 기울임, 취소선, 코드, 링크, 이미지, 표, 리스트 |
| .md 파일 열기·저장 | P0 | 웹: File System Access API. 앱: Tauri FS |
| **탭 기반 다중 문서** | P0 | Zustand 탭 스토어, 탭 생성·전환·닫기, 중복 방지 |
| 미저장 변경사항 경고 | P0 | 탭 닫기·앱 종료 시 확인 다이얼로그 |
| 자동 저장 (30초) | P1 | 변경 감지 후 로컬 자동 저장 |
| 다른 이름으로 저장 | P1 | 새 파일명·위치로 저장 |
| PDF 내보내기 버튼 | P1 | PDF 도메인 변환 기능 연결 |
| 다크/라이트/시스템 테마 전환 | P1 | next-themes 기반 테마 전환 (원래 Phase 2 예정 → Phase 1 선행 구현) |

### Phase 1 완료 기준

- [x] /editor 접속 시 Milkdown WYSIWYG 에디터 렌더링
- [x] .md 파일 열기/저장 (File System Access API + 폴백)
- [x] 탭 기반 다중 문서 관리 (Zustand tabStore)
- [x] 미저장 변경사항 경고 다이얼로그 + beforeunload
- [x] 자동 저장 30초 디바운스 (useAutoSave 훅)
- [x] PDF 내보내기 버튼 (EditorToolbar → markdownToPdf, 한글 지원)
- [x] 다크/라이트/시스템 테마 전환 (next-themes, ThemeToggle 컴포넌트)
- [x] 단위 테스트 47개 전체 통과 (`npm run test`)

### 테스트 기준

| 테스트 유형 | 범위 | 성공 기준 |
|-----------|------|----------|
| **단위 테스트** | TabStore(Zustand): openTab, closeTab, switchTab, 중복 탭 방지 로직 | 탭 동작 모든 케이스 커버. 중복 파일 경로 탭 생성 안 됨 확인 |
| **단위 테스트** | 자동 저장 타이머 로직: 30초 디바운스, 저장 성공/실패 상태 | 30초 후 저장 트리거 확인. 저장 중 변경 시 재시작 확인 |
| **통합 테스트** | Milkdown 편집기 + 탭 스토어: 탭 전환 시 각 탭 내용 독립 유지 | 탭 A 편집 후 탭 B로 전환 → 탭 A 내용 보존 확인 |
| **통합 테스트** | 파일 열기 → 탭 생성: File System Access API 모킹으로 파일 열기 테스트 | 파일 열기 성공 → 탭 생성, 파일명 표시 확인 |
| **E2E 테스트** | 전체 편집 플로우: 새 탭 생성 → 내용 입력 → 저장 → 재열기 | 저장 후 재열기 시 내용 동일 확인 |
| **E2E 테스트** | 탭 관리 플로우: 파일 2개 열기 → 탭 전환 → 탭 닫기 (미저장 경고) | 미저장 탭 닫기 시 확인 다이얼로그 표시 확인 |

## Phase 2: 편집기 완성도 강화 - 웹 (목표: Phase 1 출시 후)

**목표**: Raw 모드·단축키·Undo/Redo·스토리지 한도 경고 등 웹 환경 편집 편의 기능을 추가하여 사용자 재방문율을 높인다.

### 기능 범위

| 기능 ID | 기능 | 우선순위 | 설명 |
|---------|------|---------|------|
| E2-1 | Undo / Redo (100단계) | P1 | ProseMirror 히스토리 플러그인 |
| E2-2 | WYSIWYG ↔ Raw 모드 전환 | P1 | 동일 파일을 Raw 마크다운으로 편집 |
| E2-3 | 키보드 단축키 | P2 | Ctrl+B/I/S/T/W/Z |
| E2-5 | 웹 스토리지 한도 경고 | P2 | IndexedDB 50MB 소프트 한도 초과 시 경고 배지 + 앱 다운로드 안내 다이얼로그. E-BR12, ED-S8 |
| - | 단어·글자 수 표시 | P2 | 상태 바 실시간 통계 |
| - | **Cloud Sync Disabled - Learn More 링크** | P2 | 배지에 보안 안내 페이지 링크 추가. ED-S9, PROPOSAL-006 |
| E2-6 | 에디터 입력 영역 시각적 구분 | P1 | 콘텐츠 영역 배경색·테두리·그림자 적용 (종이 느낌). 다크/라이트 모드 대응. ED-S10 |
| E2-7 | PDF 내보내기 파일명/경로 선택 | P1 | DownloadDialog + showSaveFilePicker 연동. 미지원 브라우저 폴백. ED-S11 |
| - | Mermaid 다이어그램 | P3 | Milkdown Mermaid 플러그인 |
| - | 찾기·바꾸기 | P3 | Ctrl+F |

### 테스트 기준

| 테스트 유형 | 범위 | 성공 기준 |
|-----------|------|----------|
| **단위 테스트** | Undo/Redo 스택: 100단계 이후 FIFO로 이전 이력 제거 | 100단계 이상 Undo 시 최초 상태까지 복원 가능 |
| **단위 테스트** | 모드 전환: WYSIWYG ↔ Raw 전환 시 마크다운 AST 보존 | 모드 전환 전후 파일 내용 동일 (마크다운 직렬화/역직렬화 정확성) |
| **통합 테스트** | 키보드 단축키: Ctrl+S 저장, Ctrl+T 새 탭, Ctrl+W 탭 닫기 | 각 단축키 동작 확인 |
| **E2E 테스트** | 편집 → Undo 반복 → 저장 → 재열기 내용 확인 | 저장 시점 내용이 재열기 후에도 동일 |
| **단위 테스트** | 스토리지 한도 계산: IndexedDB 사용량 50MB 초과 시 경고 상태 반환 | 49MB: 정상, 51MB: 경고 상태 반환 확인 |

## Phase 3: 편집기 완성도 강화 - Tauri 전용 (목표: Phase 2 이후)

**목표**: Tauri 데스크탑 앱 전용 기능을 추가하여 앱 환경에서의 편집 안정성을 확보한다.

### 기능 범위

| 기능 ID | 기능 | 우선순위 | 설명 |
|---------|------|---------|------|
| E3-1 | 세션 백업 시스템 (Tauri 전용) | P1 | `{APP_DATA_DIR}/backup/` 에 탭별 .bak 파일 자동 저장. 5초 debounce. 앱 재시작 시 session.json으로 탭 복원. E-BR11, ED-S7 |

### 테스트 기준

| 테스트 유형 | 범위 | 성공 기준 |
|-----------|------|----------|
| **단위 테스트** | 세션 백업 로직: 5초 debounce 후 .bak 파일 생성, 명시적 저장 후 .bak 삭제 | .bak 파일 생성/삭제 타이밍 정확성 확인 |
| **통합 테스트** | 세션 복원: session.json 존재 시 앱 재시작 후 탭 자동 복원 | 복원된 탭 내용이 백업 .bak 파일 내용과 일치 |
| **E2E 테스트** | 앱 비정상 종료 시나리오: 편집 후 강제 종료 → 재시작 → 탭 복원 확인 | 복원된 탭에 마지막 .bak 저장 시점의 내용 표시 |

### Phase 3 태스크 정의 (shrimp-task-manager용)

| 태스크 ID | 태스크명 | PRD 근거 | 의존성 | 테스트 기준 |
|----------|---------|---------|-------|------------|
| E3-1 | 세션 백업 시스템 (Tauri 전용) | ED-S7 | Phase 2 완료 | .bak 파일 생성/삭제 타이밍 정확성, 앱 재시작 후 탭 복원 확인 |
