## 1. 도메인 개요

### 1.1 도메인 목적

Monetization 도메인은 ModuMark의 **지속 가능한 무료 서비스 운영을 위한 수익 기반**을 담당한다. Google AdSense를 유일한 수익원으로 하여, 사용자 경험을 훼손하지 않는 범위 내에서 광고를 통한 수익을 창출한다.

### 1.2 연결된 비즈니스 문제

| 전체 BRD 문제 ID | 문제 | Monetization 도메인 연관성 |
|-----------------|------|--------------------------|
| - | 1인 개발 체제의 지속 운영 비용 | 광고 수익으로 서버·인증서 비용 충당, 지속 운영 가능성 확보 |
| P2 | 품질 있는 도구가 유료 | **무료 유지를 위한 수익 모델**: AdSense 수익이 무료 서비스 운영의 재정 기반 |

## 3. 이해관계자 (도메인 레벨)

| 이해관계자 | 관심사 | 요구사항 |
|-----------|--------|---------|
| 프로젝트 오너 | 지속 운영 가능한 수익 창출 | AdSense 월 수익 목표 달성, 정책 위반 없음 |
| Google AdSense | 광고 정책 준수, 트래픽 품질 | 콘텐츠 충분성, 클릭 품질, 정책 위반 없음 |
| 최종 사용자 | 광고로 인한 기능 방해 없음 | 핵심 편집 영역에 광고 미노출, 빠른 페이지 로딩 |

> **보안 원칙**: 광고 스크립트(Google AdSense)와 파일 처리 로직은 완전히 독립적이다. 광고 스크립트는 사용자의 파일 내용에 접근할 수 없으며, 파일 처리가 서버를 거치지 않는 로컬 우선 아키텍처는 광고 통합과 무관하게 유지된다.

## 4. 핵심 비즈니스 규칙

| 규칙 ID | 규칙 | 근거 |
|---------|------|------|
| M-BR1 | 서비스는 완전 무료로 제공하며, Google AdSense가 유일한 수익원이다 | 사용자 신뢰 확보, Typora 유료화 반면교사 |
| M-BR2 | 성인 광고, 도박, 불법 콘텐츠 등 부적절한 광고 카테고리는 AdSense 정책 설정으로 차단한다 | 브랜드 이미지 보호, 사용자 신뢰 유지 |
| M-BR3 | 광고는 편집 영역(에디터 캔버스, PDF 뷰어)에 오버레이되거나 방해되지 않는다 | 핵심 기능 UX 보호 |
| M-BR4 | 광고 로딩이 Core Web Vitals(LCP, CLS)에 부정적인 영향을 미치지 않도록 lazy loading을 적용한다 | SEO 점수 유지, AdSense 승인 조건 |
| M-BR5 | 광고는 웹 버전에서만 노출된다. 데스크탑 앱(Tauri)에서는 광고가 표시되지 않는다 | 앱 스토어 정책, 데스크탑 앱 UX 최우선 |
| M-BR6 | 모바일 화면(375px 미만)에서는 광고를 숨겨 핵심 콘텐츠 접근성을 보호한다 | 모바일 UX 보호, 소형 화면 가독성 |
| M-BR7 | 광고 차단기(Ad Blocker) 사용 시 강제 차단이나 접근 거부는 하지 않으며, 비폭력적인 안내 메시지만 표시할 수 있다 | 사용자 자율성 존중, 반감 방지 |
| M-BR8 | 광고 스크립트는 사용자 파일 데이터에 접근하지 않으며, 파일 처리 로직과 광고 스크립트는 완전히 분리된다 | 보안 원칙 준수, 사용자 신뢰 |
| M-BR9 | AdSense 승인 요건 충족을 위해 각 공개 페이지는 충분한 오리지널 텍스트 콘텐츠를 포함한다 | AdSense 승인 조건, SEO 연계 |

## 5. 성공 지표 (KPI)

| KPI | 정의 | 목표 |
|-----|------|------|
| AdSense 월 수익 | Google AdSense를 통한 월간 광고 수익 | $200 (출시 6개월) |
| RPM (Revenue Per Mille) | 페이지뷰 1,000회당 수익 | $1.5 이상 유지 |
| 광고 뷰어빌리티 | 화면에 실제 표시된 광고 비율 (IAB 기준) | 50% 이상 |
| 광고 CTR | 광고 노출 대비 클릭 비율 | 0.5% 이상 |
| CLS 증가 | 광고 로딩으로 인한 레이아웃 이동 | CLS 0.1 이하 유지 |
| 광고로 인한 이탈율 변화 | 광고 도입 전후 페이지 이탈율 차이 | 5% 이내 |

## 6. 범위 정의

### 6.1 포함 범위 (In-Scope)

| 기능 | 설명 |
|------|------|
| Google AdSense 통합 | AdSense 스크립트 로딩, 광고 슬롯 컴포넌트 구현 |
| 광고 배치 전략 | 헤더 배너, 사이드바, 콘텐츠 하단 광고 슬롯 |
| 광고 정책 설정 | 부적절한 광고 카테고리 차단 (AdSense 정책 센터에서 설정) |
| Lazy Loading 적용 | 광고 슬롯 지연 로딩으로 CWV 영향 최소화 |
| Ad Blocker 감지 | 광고 차단기 사용 여부 감지 후 비폭력적 안내 |
| 광고 없는 데스크탑 앱 | Tauri 앱에서 광고 컴포넌트 미노출 처리 |
| AdSense 승인 콘텐츠 준비 | 소개 페이지, 사용법 가이드 등 오리지널 콘텐츠 |

### 6.2 제외 범위 (Out-of-Scope)

| 제외 항목 | 이유 |
|----------|------|
| 프리미엄 구독 (유료 요금제) | 무료 서비스 원칙 |
| 직접 광고 영업 (스폰서십) | 1인 운영 리소스 부족, AdSense로 충분 |
| Affiliate 마케팅 | 초기 MVP 범위 초과 |
| 광고 A/B 테스트 도구 | Phase 2 이후 검토 |
| 자체 광고 서버 운영 | 개발 복잡도 과다 |

## 7. 제약 조건

| 제약 | 내용 |
|------|------|
| AdSense 정책 | Google AdSense 콘텐츠 정책 준수 필수 (불충분한 콘텐츠, 클릭 유도 금지) |
| AdSense 승인 | 충분한 콘텐츠와 트래픽 확보 후 승인 신청 필요 (최소 3개월 콘텐츠 운영 권장) |
| CWV 제약 | 광고 스크립트가 LCP, CLS에 부정적 영향을 미치지 않도록 구현 |
| 데스크탑 앱 제약 | Tauri 앱에서는 광고 미노출 (앱 스토어 정책 및 UX) |

## 8. 리스크

| 리스크 | 영향 | 대응 |
|--------|------|------|
| Google AdSense 심사 거절 | 수익 창출 지연, 운영 비용 미충당 | 충분한 콘텐츠 확보 후 신청, 거절 시 대안 광고 네트워크(Kakao AdFit 등) 검토 |
| 광고로 인한 CWV 저하 | SEO 순위 하락, AdSense 승인 지연 | 광고 lazy loading 적용, Lighthouse 주기적 모니터링 |
| 광고 수익이 운영 비용 미달 | 서비스 지속 운영 어려움 | 초기 SEO 트래픽 확보에 집중, 트래픽 증가 시 자연스럽게 수익 증가 |
| 광고 차단기 사용률 증가 | 실제 수익이 예상보다 낮음 | 광고 차단기 의존 없이 SEO 트래픽 확대로 대응 |
