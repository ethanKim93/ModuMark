## Phase 1: PDF 핵심 기능 ✅ 완료 (목표: 출시 기반)

**목표**: PDF 뷰어·병합·분할·마크다운→PDF 변환을 완성하여 알PDF 기본 기능을 대체한다. 모든 처리는 클라이언트에서 수행하여 보안 원칙을 충족한다.

### 기능 범위

| 기능 | 우선순위 | 설명 |
|------|---------|------|
| PDF 뷰어 (PDF.js) | P0 | 페이지 이동, 줌, 페이지 번호 표시 |
| 마크다운 → PDF 변환 | P0 | 현재 문서 → PDF. 헤딩·코드블록·표 스타일 포함 |
| PDF 병합 (pdf-lib) | P0 | 최대 20개, 합산 100MB 이하 |
| PDF 분할 | P0 | 페이지 범위 추출 → 새 PDF |
| 처리 진행률 표시 | P1 | 프로그레스 바 + 단계별 텍스트 |
| 파일 크기 제한 안내 | P1 | 제한 초과 시 명확한 오류 메시지 |

### 완료 기준 (Phase 1)

- [x] PDF 뷰어 (PDF.js, 페이지 이동·줌)
- [x] 마크다운 → PDF 변환 (NotoSansKR 한글 지원)
- [x] PDF 병합 (최대 20개, 합산 100MB 이하)
- [x] PDF 분할 (페이지 범위 추출)
- [x] 처리 진행률 표시
- [x] 파일 크기 제한 안내 및 오류 메시지

| 테스트 유형 | 범위 | 성공 기준 |

## Phase 2: OCR + PDF 압축 ✅ 완료 (웹, 목표: Phase 1 안정화 후)

**목표**: Tesseract.js 기반 한국어 OCR과 PDF 압축 기능을 추가한다 (웹 환경). 모든 처리는 클라이언트에서 수행하여 기밀 문서도 안전하게 처리한다.

### 기능 범위

| 기능 ID | 기능 | 우선순위 | 설명 |
|---------|------|---------|------|
| PF2-1 | PDF OCR (Tesseract.js) | P0 | 스캔·이미지 PDF 텍스트 추출. 한국어 언어팩 |
| PF2-2 | OCR 신뢰도 표시 | P1 | 80% 미만 시 LOW_CONFIDENCE 경고 |
| PF2-3 | OCR 결과 → 에디터 탭 | P1 | 추출 텍스트를 새 탭에 마크다운으로 열기 |
| PF2-4 | OCR 언어팩 캐싱 | P2 | 초기 다운로드 후 브라우저 캐시에 저장 |
| PF2-5 | PDF 뷰어 고해상도 렌더링 (DPR) | P1 | devicePixelRatio 기반 renderScale 적용. PDF-M1 보강 |
| PF2-6 | PDF 뷰어 전용 모드 | P1 | 편집·뷰어·OCR 모드 탭 전환. PDF-S5 |
| PF2-7 | **PDF 압축 (Compress PDF)** | P1 | pdf-lib 이미지 압축 + 폰트 서브셋. 압축 레벨 선택 UI. PROPOSAL-002 채택 (PDF-S4) |

### 테스트 기준

| 테스트 유형 | 범위 | 성공 기준 |
|-----------|------|----------|
| **단위 테스트** | Tesseract.js Worker 초기화·종료 사이클 | Worker 메모리 누수 없음. 초기화 후 재사용 가능 |
| **단위 테스트** | OCR 신뢰도 계산: 신뢰도 80% 미만 케이스 정확히 LOW_CONFIDENCE 반환 | 경계값(79%, 80%, 81%) 테스트 |
| **통합 테스트** | OCR 파이프라인: PDF → PDF.js 페이지 렌더링 → Canvas → Tesseract.js 텍스트 추출 | 10페이지 PDF OCR 완료, 결과 텍스트 반환 |
| **E2E 테스트** | OCR 플로우: 스캔 PDF 업로드 → OCR 시작 → 진행률 → 결과 에디터 탭으로 전달 | OCR 완료 후 에디터 탭에 추출 텍스트 표시 |
| **E2E 테스트** | 한국어 OCR 정확도: 표준 한국어 인쇄 문서 샘플로 정확도 측정 | 80% 이상 정확도 달성 |
| **단위 테스트** | PDF 압축 로직: 이미지 압축 전후 파일 크기 비교 | 압축 후 파일 크기가 원본 대비 감소 확인 |
| **E2E 테스트** | PDF 압축 플로우: 파일 업로드 → 압축 레벨 선택 → 압축 → 다운로드 | 다운로드 파일이 원본보다 작고 PDF 구조 유효성 확인 |

- [x] OCR 신뢰도 표시 (80% 미만 LOW_CONFIDENCE)

- [x] OCR 언어팩 IndexedDB 캐싱

## Phase 4: 단일 뷰어 + 파일 탭 + 액션 버튼 아키텍처

**목표**: 모드 탭을 완전 제거하고, 파일 탭 + 액션 버튼 다이얼로그 방식의 새 UI로 전면 재구성한다. 뷰어가 유일한 화면이 되며, 모든 작업(분할·압축·OCR·병합)은 다이얼로그 오버레이를 통해 실행한다.

### 기능 범위

| 기능 ID | 기능 | 설명 |
|---------|------|------|
| PF4-1 | 모드 탭 제거 + 뷰어 단일화 | PdfEditor의 mode 상태 및 모드 탭 UI 완전 제거. 뷰어가 유일한 화면 |
| PF4-2 | 파일 탭 UI 구현 | 여러 PDF를 열면 상단에 브라우저 탭처럼 표시. 탭 전환, 닫기, 활성 탭 강조 |
| PF4-3 | 액션 버튼 툴바 구현 | 뷰어 상단에 [분할] [압축] [OCR] [파일 추가] 버튼 고정 배치 |
| PF4-4 | 분할 다이얼로그 오버레이 | [분할] 버튼 → 다이얼로그에서 페이지 범위 입력 → 분할 실행 → 다운로드 |
| PF4-5 | 압축 다이얼로그 오버레이 | [압축] 버튼 → 다이얼로그에서 레벨 선택 → 압축 실행 → 다운로드 |
| PF4-6 | OCR 다이얼로그 오버레이 | [OCR] 버튼 → 다이얼로그에서 언어 선택 → OCR 실행 → 에디터 탭으로 전달 |
| PF4-7 | 병합 인라인 흐름 | [파일 추가] → 추가 파일 페이지가 뷰어 하단에 인라인으로 붙음 → [병합 저장] |
| PF4-8 | pdfFileStore 파일 탭 상태 관리 | 여러 PDF의 독립적 뷰어 상태(currentPage, zoom 등)를 탭별로 관리 |

## Phase 5: 통합 PDF 사이드바 — 뷰어 네비게이션 + 병합 파일 목록

**목표**: 좌측 사이드바를 단일 패널로 통합하여 뷰어 페이지 썸네일 네비게이션(상단)과 병합 파일 목록(하단)을 하나의 사이드바에서 제공한다. 사용자가 뷰어 화면을 벗어나지 않고 페이지 이동·병합 파일 관리·병합 실행을 모두 수행할 수 있다.

**PRD 근거**: PDF-S10(FR-S1)·PDF-S11(FR-S2)·PDF-S12(FR-S3)·PDF-S13(FR-S4)·PDF-S14(FR-S5), BRD P-BR11, P-G6

**의존 Phase**: Phase 4 (단일 뷰어 + 파일 탭 + 액션 버튼 아키텍처)

### 기능 범위

| 기능 ID | 기능 | 우선순위 | 설명 |
|---------|------|---------|------|
| PF5-1 | 통합 사이드바 스토어 | P0 | pdfFileStore에 통합 사이드바 상태 추가 — 현재 페이지 인덱스, 병합 파일 목록, 사이드바 너비 |

### 기능 구조

```
UnifiedPdfSidebar.tsx (최상위 컴포넌트)
├── PdfSidebarPageNav.tsx       ← 상단: 페이지 썸네일 네비게이션
│   ├── generateThumbnail.ts   (기존 lib 재사용)
│   └── 썸네일 클릭 → onPageChange 콜백 → PdfEditor 페이지 이동
├── [수평 divider]
└── PdfSidebarMergeList.tsx    ← 하단: 병합 파일 목록
    ├── dnd-kit SortableContext (파일 순서 드래그)
    ├── 빈 상태: 드롭존 (점선 테두리 + 안내 문구)
    ├── 파일 항목: 파일명 + 페이지 수 + 제거(X) 버튼
    └── [병합 저장] 버튼 (고정 하단, 파일 1개 이상 시 활성화)
```

## Phase 6: PDF 고급 변환 (향후 검토)

> **Won't Have (명시)**: PDF to Word 변환은 PROPOSAL-003 기각 결정으로 구현하지 않음. 정확한 레이아웃 재현이 기술적으로 어려움.

**목표**: PDF → 마크다운 변환으로 PDF 문서를 재편집 가능한 마크다운으로 가져올 수 있게 한다.

### 기능 범위 (검토 중)

| 기능 | 설명 |
|------|------|
| PDF → 마크다운 변환 | PDF 텍스트 레이어 파싱 → 마크다운 구조 변환 |
| PDF 암호 설정·해제 | pdf-lib 암호화 기능 활용 |

### 테스트 기준

| 테스트 유형 | 범위 |
|-----------|------|
| **단위 테스트** | PDF 텍스트 레이어 파싱 정확도, 마크다운 구조 변환 규칙 |
| **통합 테스트** | 변환 파이프라인 전체 흐름 |
| **E2E 테스트** | 사용자 시나리오 전체 + 기존 기능 회귀 테스트 |
