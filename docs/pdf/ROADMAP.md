## Phase 1: PDF 핵심 기능 (목표: 출시 기반)

**목표**: PDF 뷰어·병합·분할·마크다운→PDF 변환을 완성하여 알PDF 기본 기능을 대체한다. 모든 처리는 클라이언트에서 수행하여 보안 원칙을 충족한다.

### 기능 범위

| 기능 | 우선순위 | 설명 |
|------|---------|------|
| PDF 뷰어 (PDF.js) | P0 | 페이지 이동, 줌, 페이지 번호 표시 |
| 마크다운 → PDF 변환 | P0 | 현재 문서 → PDF. 헤딩·코드블록·표 스타일 포함 |
| PDF 병합 (pdf-lib) | P0 | 최대 20개, 합산 100MB 이하 |
| PDF 분할 | P0 | 페이지 범위 추출 → 새 PDF |
| 처리 진행률 표시 | P1 | 프로그레스 바 + 단계별 텍스트 |
| 파일 크기 제한 안내 | P1 | 제한 초과 시 명확한 오류 메시지 |

### 테스트 기준

| 테스트 유형 | 범위 | 성공 기준 |
|-----------|------|----------|
| **단위 테스트** | pdf-lib 병합 로직: 2개 이상 PDF 병합 시 페이지 순서·수 정확성 | 병합된 PDF의 총 페이지 수 = 각 파일 페이지 수의 합 |
| **단위 테스트** | pdf-lib 분할 로직: 페이지 범위 1-3, 5, 7-10 추출 정확성 | 추출된 PDF의 페이지 수 및 내용이 원본과 일치 |
| **단위 테스트** | 파일 크기 검증: 100MB 초과 파일 처리 시도 시 오류 반환 | 크기 초과 파일에 ValidationError 반환 확인 |
| **통합 테스트** | 마크다운 → PDF 변환: 에디터 탭 내용 → PDF 생성 → Blob 다운로드 | 생성된 PDF 열기 가능, 헤딩·코드블록 스타일 포함 |
| **통합 테스트** | PDF.js 뷰어: 100페이지 PDF 페이지 이동·줌 정상 동작 | 페이지 이동 지연 500ms 이내, 줌 정확도 확인 |
| **E2E 테스트** | 전체 병합 플로우: 파일 3개 선택 → 순서 지정 → 병합 → 다운로드 | 다운로드 파일이 올바른 페이지 수와 순서를 가짐 |
| **E2E 테스트** | 전체 분할 플로우: PDF 열기 → 페이지 범위 입력 → 분할 → 다운로드 | 지정 페이지만 포함된 PDF 다운로드 |

## Phase 2: OCR + PDF 압축 추가 (목표: Phase 1 안정화 후)

**목표**: Tesseract.js 기반 한국어 OCR과 PDF 압축 기능을 추가한다. 모든 처리는 클라이언트에서 수행하여 기밀 문서도 안전하게 처리한다.

### 기능 범위

| 기능 | 우선순위 | 설명 |
|------|---------|------|
| PDF OCR (Tesseract.js) | P0 | 스캔·이미지 PDF 텍스트 추출. 한국어 언어팩 |
| OCR 신뢰도 표시 | P1 | 80% 미만 시 LOW_CONFIDENCE 경고 |
| OCR 결과 → 에디터 탭 | P1 | 추출 텍스트를 새 탭에 마크다운으로 열기 |
| OCR 언어팩 캐싱 | P2 | 초기 다운로드 후 브라우저 캐시에 저장 |
| **PDF 압축 (Compress PDF)** | P1 | pdf-lib 이미지 압축 + 폰트 서브셋. 압축 레벨 선택 UI. PROPOSAL-002 채택 (PDF-S4) |

### 테스트 기준

| 테스트 유형 | 범위 | 성공 기준 |
|-----------|------|----------|
| **단위 테스트** | Tesseract.js Worker 초기화·종료 사이클 | Worker 메모리 누수 없음. 초기화 후 재사용 가능 |
| **단위 테스트** | OCR 신뢰도 계산: 신뢰도 80% 미만 케이스 정확히 LOW_CONFIDENCE 반환 | 경계값(79%, 80%, 81%) 테스트 |
| **통합 테스트** | OCR 파이프라인: PDF → PDF.js 페이지 렌더링 → Canvas → Tesseract.js 텍스트 추출 | 10페이지 PDF OCR 완료, 결과 텍스트 반환 |
| **E2E 테스트** | OCR 플로우: 스캔 PDF 업로드 → OCR 시작 → 진행률 → 결과 에디터 탭으로 전달 | OCR 완료 후 에디터 탭에 추출 텍스트 표시 |
| **E2E 테스트** | 한국어 OCR 정확도: 표준 한국어 인쇄 문서 샘플로 정확도 측정 | 80% 이상 정확도 달성 |
| **단위 테스트** | PDF 압축 로직: 이미지 압축 전후 파일 크기 비교 | 압축 후 파일 크기가 원본 대비 감소 확인 |
| **E2E 테스트** | PDF 압축 플로우: 파일 업로드 → 압축 레벨 선택 → 압축 → 다운로드 | 다운로드 파일이 원본보다 작고 PDF 구조 유효성 확인 |

## Phase 3: PDF 고급 변환 (향후 검토)

> **Won't Have (명시)**: PDF to Word 변환은 PROPOSAL-003 기각 결정으로 구현하지 않음. 정확한 레이아웃 재현이 기술적으로 어려움.

**목표**: PDF → 마크다운 변환으로 PDF 문서를 재편집 가능한 마크다운으로 가져올 수 있게 한다.

### 기능 범위 (검토 중)

| 기능 | 설명 |
|------|------|
| PDF → 마크다운 변환 | PDF 텍스트 레이어 파싱 → 마크다운 구조 변환 |
| PDF 페이지 미리보기 | 병합·분할 전 썸네일 선택 UI |
| PDF 암호 설정·해제 | pdf-lib 암호화 기능 활용 |

### 테스트 기준

| 테스트 유형 | 범위 |
|-----------|------|
| **단위 테스트** | PDF 텍스트 레이어 파싱 정확도, 마크다운 구조 변환 규칙 |
| **통합 테스트** | 변환 파이프라인 전체 흐름 |
| **E2E 테스트** | 사용자 시나리오 전체 + 기존 기능 회귀 테스트 |
