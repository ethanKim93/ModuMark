## 1. 도메인 개요

### 1.1 도메인 목적

PDF 도메인은 ModuMark의 차별화 기능인 **무료 PDF 처리 통합**을 담당한다. 마크다운과 PDF를 단일 앱에서 처리함으로써, 별도 도구(알PDF, Adobe)를 쓰던 사용자가 ModuMark 하나로 모든 문서 작업을 완결할 수 있게 한다.

### 1.2 연결된 비즈니스 문제

| 전체 BRD 문제 ID | 문제 | PDF 도메인 연관성 |
|-----------------|------|-----------------|
| P1 | 마크다운 에디터와 PDF 편집 도구 분리 | **직접 해결**: 동일 앱 내 PDF 처리 |
| P3 | PDF 편집·OCR·변환 도구가 마크다운과 미연동 | **직접 해결**: 마크다운↔PDF 상호 변환 |

## 3. 이해관계자 (도메인 레벨)

| 이해관계자 | 관심사 | 요구사항 |
|-----------|--------|---------|
| 학생 (이수진) | 스캔 PDF에서 텍스트 추출, 마크다운 → PDF 제출 | OCR 정확도, 원클릭 PDF 변환 |
| 사무직 (박지현) | PDF 병합·분할, 무료 PDF 도구 대체 | 알PDF 수준의 병합·분할 기능 |
| 개발자 (김민준) | 마크다운 → PDF 변환, 문서 공유 | 코드 블록 포함 PDF 품질 |
| 보안 민감 기업 사용자 (최성호) | 계약서·기획안 등 기밀 PDF가 외부 서버에 업로드되지 않아야 함 (iLovePDF, Smallpdf 대체 수요가 가장 큰 영역) | **클라이언트 전용 PDF 처리**: OCR 포함 모든 PDF 처리를 브라우저/앱 내에서 수행, 파일 외부 전송 없음 |

## 4. 핵심 비즈니스 규칙

| 규칙 ID | 규칙 | 근거 |
|---------|------|------|
| P-BR1 | 처리되는 PDF 파일은 어떠한 경우에도 외부 서버로 전송되거나 저장되지 않는다. OCR 포함 모든 PDF 처리는 클라이언트(브라우저 또는 데스크탑 앱) 내에서만 수행된다 | 로컬 우선 원칙, 개인정보보호법, 기업 보안 정책 준수 (보안 페르소나: 최성호) |
| P-BR2 | 서버리스 처리 시 단일 파일 크기는 5MB 이하로 제한한다 | Vercel 무료 티어 제약 (BRD 8.1) |
| P-BR3 | 병합 대상 파일은 최대 20개, 총 합산 100MB 이하로 제한한다 | 메모리·성능 안정성 |
| P-BR4 | OCR 처리는 우선적으로 클라이언트(Tesseract.js)에서 수행한다 | 서버 비용 최소화, 개인정보 보호 |
| P-BR5 | OCR 결과 신뢰도가 80% 미만인 경우 LOW_CONFIDENCE 경고를 표시한다 | 사용자에게 품질 정보 제공 |
| P-BR6 | PDF 변환 결과는 사용자의 로컬 파일 시스템에만 저장한다 | 로컬 우선 원칙 |
| P-BR7 | PDF 압축 기능을 제공한다. pdf-lib 이미지 압축 + 폰트 서브셋 방식으로 파일 크기를 줄이며, 모든 처리는 클라이언트에서 수행한다 | 알PDF 대체 완성도 향상, PROPOSAL-002 채택 (Phase 2) |

| KPI | 정의 | 목표 |
|-----|------|------|
| MD→PDF 변환 성공률 | 변환 요청 대비 성공 비율 | 99% 이상 |
| PDF 병합 성공률 | 병합 요청 대비 성공 비율 | 99% 이상 |

| PDF 기능 주간 사용 횟수 | 주간 PDF 기능 사용 횟수 합계 | 1,000회 이상 (출시 3개월 후) |

## 6. 범위 정의

### 6.1 포함 범위 (In-Scope)

| 기능 | 설명 |
|------|------|
| PDF 뷰어 | PDF 파일 렌더링, 페이지 이동, 줌 |
| PDF 병합 | 여러 PDF를 하나로 합치기 (최대 20개) |
| PDF 분할 | 특정 페이지 범위 추출하여 새 PDF 생성 |
| 마크다운 → PDF | 현재 편집 중인 마크다운 문서를 PDF로 변환 |
| PDF OCR | 스캔·이미지 PDF에서 텍스트 추출 (Tesseract.js) |
| PDF 압축 | PDF 파일 크기 압축 (이미지 압축 + 폰트 서브셋, Phase 2) |
| PDF → 마크다운 | PDF 텍스트 레이어를 마크다운으로 변환 |
| 진행률 표시 | 처리 중인 작업의 진행 상태 표시 |

### 6.2 제외 범위 (Out-of-Scope)

| 제외 항목 | 이유 |
|----------|------|
| PDF 직접 편집 (텍스트 수정) | 구현 복잡도 과다, PDF 구조 특성상 어려움 |
| PDF 전자서명 | 법적 요구사항 복잡 |
| PDF 양식(Form) 작성 | MVP 범위 초과 |
| PDF 암호 설정/해제 | Phase 2 이후 검토 |
| 5MB 초과 파일 서버 처리 | Vercel 무료 티어 제약 |

## 7. 제약 조건

| 제약 | 내용 |
|------|------|
| 서버리스 제한 | Vercel 함수 실행 10초 제한, 페이로드 5MB 제한 |
| OCR 정확도 | Tesseract.js 한국어 모델 의존 (A3 가정) |
| 클라이언트 메모리 | 대용량 PDF 처리 시 브라우저 메모리 한계 |

## 8. 리스크

| 리스크 | 영향 | 대응 |
|--------|------|------|
| OCR 한국어 정확도 부족 | 핵심 기능 신뢰도 저하 | 출시 전 정확도 벤치마크, 80% 미달 시 베타 기능으로 표기 |
