## 1. 기술 선정

### 1.1 pdf-lib (PDF 생성·편집)

**선택 근거**:
- 순수 TypeScript/JavaScript 구현. 서버 의존 없이 브라우저에서 완전히 동작
- 보안 원칙(로컬 처리, 파일 서버 전송 없음) 완벽 충족
- PDF 병합, 분할, 생성, 페이지 조작 등 핵심 기능 지원
- 활발한 오픈소스 유지보수 (MIT 라이선스)

**한계**: 복잡한 PDF 레이아웃(CSS 기반 스타일 변환)은 제한적. 마크다운 → PDF 변환 시 스타일 제어에 추가 작업 필요.

### 1.2 PDF.js (PDF 뷰어)

**선택 근거**:
- Mozilla 재단 공식 오픈소스 PDF 렌더링 엔진
- 브라우저 내 PDF 렌더링의 사실상 표준
- Canvas 기반 렌더링으로 높은 호환성
- 페이지 이동, 줌, 검색 등 뷰어 기능 내장
- 웹 환경에서 검증된 안정성

### 1.3 Tesseract.js (OCR)

**선택 근거**:
- Tesseract OCR 엔진의 WebAssembly 포팅. 브라우저에서 직접 실행
- **서버 전송 없이 클라이언트에서 OCR 수행** → 기업 보안 정책 준수 핵심
- 한국어(`kor`) 언어팩 지원. 한국어 OCR 사용 가능
- 오픈소스 (Apache 2.0)

**한계**: 서버 사이드 Tesseract 대비 초기화 시간이 길고(언어팩 다운로드), 처리 속도가 느릴 수 있음. 초기 로딩 시 한국어 언어팩 캐싱 필요.

**검증 필요 사항**: 한국어 표준 문서 OCR 정확도 80% 이상 달성 여부를 출시 전 테스트.

## 2. 기능 요구사항 (MoSCoW)

### Must Have

| 기능 ID | 기능 | 설명 |
|---------|------|------|
| PDF-M1 | PDF 뷰어 | PDF.js 기반. 페이지 이동, 줌인/줌아웃, 페이지 번호 표시. 고해상도 렌더링(devicePixelRatio 대응) — **✅ Phase 1+2A 구현 완료 (DPR 고해상도 포함)** |
| PDF-M2 | 마크다운 → PDF 변환 | 현재 편집 중인 마크다운 문서를 PDF로 변환·다운로드. 헤딩·코드블록·표 스타일 포함. 파일명 입력 + 저장 경로 선택 지원 |
| PDF-M3 | PDF 병합 | 여러 PDF 파일을 하나로 합치기. 최대 20개, 합산 100MB 이하 |
| PDF-M4 | PDF 분할 | 지정한 페이지 범위를 추출하여 새 PDF 파일 생성 |
| PDF-M5 | 처리 진행률 표시 | 병합·분할·OCR 작업 진행 상태 프로그레스 바 표시 |

|---------|------|------|
| PDF-S1 | PDF OCR | Tesseract.js 기반. 스캔·이미지 PDF에서 텍스트 추출. 한국어 언어팩 포함 — **✅ Phase 2A 구현 완료** |
| PDF-S2 | OCR 신뢰도 경고 | OCR 결과 신뢰도 80% 미만 시 LOW_CONFIDENCE 경고 표시 |
| PDF-S3 | 파일 크기 제한 안내 | 단일 파일 처리 한계 초과 시 명확한 안내 메시지 |
| PDF-S4 | PDF 압축 (Compress PDF) | pdf-lib 이미지 품질 저하 압축 + 폰트 서브셋 적용으로 파일 크기 축소. 압축 레벨 선택 UI 제공. 모든 처리 클라이언트에서 수행. PROPOSAL-002 채택 — **✅ Phase 2A 구현 완료** |

| PDF-S13 | 통합 사이드바 - 너비 조절 (FR-S4) | 사이드바 오른쪽 경계를 드래그하여 너비 조절 가능. 최소 200px, 최대 400px. 조절된 너비는 세션 동안 유지. Phase 3 |
| PDF-S14 | 통합 사이드바 - 병합 실행 버튼 (FR-S5) | 병합 파일 목록 하단 고정 위치에 [병합 저장] 버튼 배치. 파일이 1개 이상일 때 활성화. 클릭 시 병합 실행 → 진행률 표시 → 다운로드. Phase 3 |

## 4. 처리 흐름 상세

### 4.1 마크다운 → PDF 변환 흐름

```
[에디터] → PDF 내보내기 버튼 클릭
    ↓
마크다운 AST를 HTML로 변환 (unified/remark)
    ↓
HTML + CSS 스타일 적용 (코드 하이라이팅, 표 스타일)
    ↓
pdf-lib로 PDF 문서 생성 (클라이언트에서 실행)
    ↓
Blob URL → 브라우저 다운로드 / Tauri FS 저장
```

### 4.2 PDF 병합 흐름

```
[사용자] → PDF 파일 여러 개 선택 (드래그 앤 드롭 또는 파일 선택)
    ↓
파일 순서 지정 (드래그 재정렬)
    ↓
pdf-lib: 각 PDF의 페이지를 새 PDFDocument에 복사 (클라이언트)
    ↓
진행률 표시 (페이지 단위)
    ↓
완료 → 다운로드
```

### 4.3 OCR 흐름

```
[사용자] → PDF 파일 선택 → OCR 시작
    ↓
PDF.js로 각 페이지를 Canvas로 렌더링
    ↓
Canvas를 이미지로 변환
    ↓
Tesseract.js Worker: 이미지 → 텍스트 (브라우저 내, 서버 전송 없음)
    ↓
신뢰도(confidence) 확인 → 80% 미만 시 LOW_CONFIDENCE 표시
    ↓
추출된 텍스트 → 새 에디터 탭으로 전달 (마크다운 형식)
```

### 4.4 단일 뷰어 + 액션 버튼 흐름 (Phase 3)

```
[사용자] → PDF 파일 드롭/선택
    ↓
뷰어로 바로 열림 (모드 탭 없음)
    ↓
상단: 파일 탭 (여러 파일 열 경우) + 액션 버튼 툴바 [분할] [압축] [OCR] [파일 추가]
    ↓
[분할]: 버튼 클릭 → 분할 다이얼로그 오버레이 → 페이지 범위 입력 → 실행 → 다운로드
[압축]: 버튼 클릭 → 압축 다이얼로그 오버레이 → 압축 레벨 선택 → 실행 → 다운로드
[OCR]:  버튼 클릭 → OCR 다이얼로그 오버레이 → 페이지 범위 선택 → 실행 → 에디터 탭 전달
    ↓
[병합]: [파일 추가] 클릭 → 추가 파일 선택
    ↓
추가 파일 페이지가 현재 뷰어 하단에 인라인으로 붙어 표시
    ↓
[병합 저장] 버튼 클릭 → 병합된 PDF 다운로드
```

**핵심 원칙**:
- 사용자는 뷰어 화면을 벗어나지 않는다
- 각 액션은 다이얼로그 오버레이(모달)로 처리되며, 뷰어 컨텍스트(파일 상태)는 유지된다
- 파일 탭 전환 시 이전 탭의 스크롤 위치·줌 레벨이 복원된다

### 4.5 통합 사이드바 흐름 (Phase 3)

```
[사용자] → PDF 파일 열기
    ↓
좌측 사이드바 상단: 현재 파일 페이지 썸네일 목록 렌더링
    ↓
[썸네일 클릭] → 뷰어 메인 영역이 해당 페이지로 스크롤
              → 클릭한 썸네일에 primary 컬러 하이라이트 표시
    ↓
[파일 추가 버튼 또는 드롭존 드롭]
    ↓
사이드바 하단에 추가된 파일 항목 표시 (파일명, 페이지 수, 제거 버튼)
    ↓
[드래그 정렬] → dnd-kit으로 파일 순서 변경
    ↓
[병합 저장 버튼 클릭] → 진행률 표시 → 병합 PDF 다운로드
```

**통합 사이드바 핵심 원칙** (BRD P-BR11, P-G6 대응):
- 사이드바 상단(페이지 네비게이션)과 하단(병합 파일 목록)은 **단일 사이드바 내 연속된 공간**이다. 별도 패널 전환 없음
- 뷰어↔병합 전환 클릭 횟수: **0회** (사이드바를 벗어나지 않음)
- 수평 divider로 두 영역을 시각적으로 구분하되, 각 영역은 독립 스크롤로 동작한다

## 5. 비기능 요구사항

### 5.1 성능

| 요구사항 | 목표 |
|---------|------|
| PDF 처리 완료 시간 (10MB 이하) | P90 기준 10초 이내 |
| MD → PDF 변환 시간 | 일반 문서(10페이지 이하) 3초 이내 |
| OCR 초기화 시간 (언어팩 로딩) | 최초 1회. 이후 캐싱으로 즉시 사용 |

### 5.2 보안 (핵심)

| 요구사항 | 구현 |
|---------|------|
| **파일 서버 전송 없음** | pdf-lib, PDF.js, Tesseract.js 모두 클라이언트에서 동작. 파일 Blob이 네트워크를 통해 외부로 전송되지 않음 |
| 메모리 보안 | 처리 완료된 파일 Blob은 사용 후 메모리에서 해제 (URL.revokeObjectURL) |
| 파일 크기 제한 | 단일 파일 최대 처리 크기를 안내하여 메모리 초과 방지 |

### 5.3 신뢰성

| 요구사항 | 목표 |
|---------|------|
| MD→PDF 변환 성공률 | 99% 이상 |
| PDF 병합 성공률 | 99% 이상 |
| 한국어 OCR 정확도 | 표준 문서 기준 80% 이상 |
| 실패 시 오류 메시지 | 구체적이고 이해하기 쉬운 오류 안내 |

## 6. UI/UX 요구사항

### 6.1 PDF 기능 진입점 (Phase 3 UX 리디자인)

- 편집기 상태 바 또는 메뉴: "PDF로 내보내기" 버튼 (현재 문서 → PDF)
- **`/pdf` 단일 뷰어 화면**: 파일 드롭/선택 시 모드 탭 없이 뷰어로 바로 열림. 기존 모드 탭(편집·뷰어·OCR) 제거
- **파일 탭 UI**: 여러 PDF를 열면 상단에 브라우저 탭 스타일로 표시. 각 탭은 파일명·닫기(X) 버튼 포함. 활성 탭 강조(primary 컬러 언더라인 또는 배경 강조). 탭 클릭으로 뷰어 전환
- **액션 버튼 툴바**: 뷰어 상단 고정. 아이콘+텍스트 버튼 형태. [분할] [압축] [OCR] [파일 추가] 4개 버튼. 처리 중 버튼 비활성화 + 로딩 스피너
- **다이얼로그 오버레이**: 각 액션 버튼 클릭 시 모달 방식으로 옵션 UI 표시. 배경 뷰어는 흐림(backdrop-blur) 처리. 다이얼로그 닫기 시 뷰어 상태 복원
- **병합 인라인 흐름**: [파일 추가] → 추가 파일 선택 → 뷰어 하단에 추가 파일 페이지가 순서대로 붙어 인라인 표시 → 구분선으로 파일 경계 표시 → [병합 저장] 버튼
- **통합 사이드바**: 좌측 고정 사이드바. 상단은 현재 열린 PDF 페이지 썸네일 네비게이션, 하단은 병합 대상 파일 목록. 수평 divider로 두 영역 구분. 너비 200px~400px 드래그 조절. 하단 고정 [병합 저장] 버튼 포함. 세부 명세는 [7. 통합 사이드바 기능 명세](#7-통합-사이드바-기능-명세) 참조

### 6.1.1 페이지 단위 관리 모델 (Phase 1 구현)

- **썸네일 선택**: `PdfThumbnailList.tsx`에서 PDF 파일의 각 페이지를 썸네일로 표시. 개별 또는 다중 선택 가능
- **dnd-kit 정렬**: 썸네일을 드래그하여 병합 순서 또는 페이지 순서 재배치 (`@dnd-kit/sortable`)
- **다중 페이지 선택**: 체크박스 또는 Shift+클릭으로 여러 페이지 동시 선택 후 추출·삭제
- **Undo 30단계**: `pdfFileStore.ts`의 히스토리 관리로 최대 30단계 실행 취소 지원 (`canUndo`, `undo()`)

### 6.2 처리 중 UX

- 처리 중 프로그레스 바 + 현재 처리 단계 텍스트 표시 ("페이지 3/10 처리 중...")
- 처리 중 취소 버튼 제공
- 처리 완료 후 즉시 다운로드 시작 또는 저장 완료 알림

### 6.3 오류 처리 UX

- 파일 크기 초과: "파일 크기가 100MB를 초과합니다. 분할하여 처리해 주세요."
- OCR 저신뢰도: "일부 텍스트의 인식 정확도가 낮습니다. 결과를 확인하고 수정해 주세요."
- 지원하지 않는 PDF: "이 PDF는 처리할 수 없습니다. (암호화된 PDF 등)"

## 7. 통합 사이드바 기능 명세

> BRD P-BR11, P-G6 대응. 뷰어 단일 화면(PDF-S6) 및 병합 인라인 흐름(PDF-S9)과 연동되는 Phase 3 신규 기능.

### 7.1 사이드바 레이아웃 구조

```
┌──────────────────────────┐
│  [상단 영역] 페이지 썸네일   │  ← 독립 스크롤 (overflow-y: auto)
│  thumb 1  [현재 페이지 강조]│
│  thumb 2                 │
│  thumb 3                 │
│  ...                     │
├──────────── divider ─────┤
│  [하단 영역] 병합 파일 목록  │  ← 독립 스크롤 (overflow-y: auto)
│  📄 file-a.pdf  (3p) [X] │
│  📄 file-b.pdf  (5p) [X] │
│  [+ 파일 추가 드롭존]       │
│                          │
│  [병합 저장] (고정 하단)    │
└──────────────────────────┘
```

### 7.2 페이지 썸네일 네비게이션 (FR-S1) 상세 명세

| 항목 | 명세 |
|------|------|
| 렌더링 방식 | PDF.js로 각 페이지를 Canvas 렌더링 → 썸네일 이미지 생성 (`generateThumbnail.ts` 활용) |
| 썸네일 크기 | 너비 사이드바 너비 - 24px 패딩. 비율 유지 |
| 현재 페이지 표시 | primary 컬러(`#1773CF`) 2px 테두리 + 배경 강조 |
| 클릭 동작 | 뷰어 메인 영역 해당 페이지로 스크롤. 300ms 이내 응답 |
| 뷰어 스크롤 동기화 | 뷰어를 수동 스크롤할 때 현재 페이지 썸네일 하이라이트 자동 업데이트 |
| 페이지 번호 표시 | 썸네일 하단에 페이지 번호 텍스트 표시 |
| 성능 | 가상 스크롤(virtualization) 적용 권장. 100페이지 이상 PDF에서 초기 렌더 50개 이내 제한 |

### 7.3 병합 파일 목록 (FR-S2) 상세 명세

| 항목 | 명세 |
|------|------|
| 파일 항목 구성 | 파일 아이콘 + 파일명(최대 20자 truncate) + 페이지 수 + 제거(X) 버튼 |
| 파일 추가 방법 | 드롭존 드래그 앤 드롭 또는 [+ 파일 추가] 버튼 클릭 → 파일 선택 다이얼로그 |
| 추가 위치 | 목록 최하단에 순서대로 추가 |
| 순서 변경 | dnd-kit `@dnd-kit/sortable` 드래그 핸들로 순서 변경 가능 |
| 제거 | 항목의 [X] 버튼 클릭 → 목록에서 즉시 제거 |
| 빈 상태 | 파일이 없을 때 점선 테두리 드롭존 + "PDF 파일을 끌어다 놓거나 클릭하여 추가하세요" 안내 문구 표시 |
| 파일 수 제한 | 최대 19개 (현재 열린 파일 1개 포함 총 20개 제한, P-BR3 대응) |
| 합산 크기 제한 | 현재 파일 + 추가 파일 합산 100MB 초과 시 추가 거부 + 안내 토스트 표시 |

### 7.4 수용 기준 (Acceptance Criteria)

| ID | 기준 |
|----|------|
| AC-S10-1 | 사이드바 상단에 현재 열린 PDF의 모든 페이지 썸네일이 표시된다 |
| AC-S10-2 | 썸네일 클릭 시 300ms 이내에 뷰어 메인이 해당 페이지로 이동한다 |
| AC-S10-3 | 현재 보이는 페이지에 해당하는 썸네일이 primary 컬러 테두리로 강조된다 |
| AC-S11-1 | 사이드바 하단에 병합 파일 목록이 표시되며, 빈 상태에서 드롭존이 표시된다 |
| AC-S11-2 | 파일 추가 시 목록 하단에 즉시 추가되며 파일명·페이지 수·제거 버튼이 표시된다 |
| AC-S11-3 | dnd-kit 드래그로 파일 순서를 변경할 수 있다 |
| AC-S11-4 | 제거(X) 버튼 클릭 시 해당 파일이 목록에서 즉시 제거된다 |
| AC-S12-1 | 두 영역 사이에 수평 divider가 표시된다 |
| AC-S12-2 | 상단과 하단 영역이 각각 독립적으로 스크롤된다 (한 영역 스크롤이 다른 영역에 영향 없음) |
| AC-S13-1 | 사이드바 오른쪽 경계를 드래그하여 너비를 200px~400px 범위에서 조절할 수 있다 |
| AC-S14-1 | 병합 파일이 1개 이상일 때 [병합 저장] 버튼이 활성화된다 |
| AC-S14-2 | [병합 저장] 버튼 클릭 시 진행률 바가 표시되고 완료 후 병합 PDF가 다운로드된다 |

### 7.5 엣지 케이스

| 케이스 | 처리 방식 |
|--------|----------|
| 단일 페이지 PDF | 썸네일 1개만 표시. 스크롤 없음 |
| 100페이지 이상 PDF | 가상 스크롤 적용. 뷰포트에 보이는 썸네일만 실제 렌더링 |
| 병합 파일 20개 초과 시도 | 추가 거부 + "최대 20개 파일까지 병합 가능합니다" 토스트 표시 |
| 합산 100MB 초과 시도 | 추가 거부 + "파일 크기 합계가 100MB를 초과합니다" 토스트 표시 |
| 동일 파일 중복 추가 | 허용 (사용자가 의도적으로 같은 파일을 여러 번 병합하는 경우 존재) |
| 사이드바 최소 너비(200px) 도달 | 더 이상 축소 불가. 경계 드래그 시 최솟값 고정 |
| 파일 탭 전환 시 | 각 파일 탭은 독립 컨텍스트 유지. 탭 전환 시 해당 파일의 썸네일 목록이 사이드바 상단을 교체함. 병합 파일 목록은 탭 공통 상태 유지 (탭별 별도 목록 vs 공통 목록은 구현 시 결정) |
