## 1. 기술 선정

### 1.1 Milkdown (WYSIWYG 에디터 코어)

**선택 근거**:
- ProseMirror 기반으로 Typora와 동일한 수준의 WYSIWYG 마크다운 경험 구현 가능
- 마크다운 파일(.md) 읽기·쓰기 완전 호환 (마크다운 AST 직접 변환)
- 플러그인 아키텍처: 헤딩, 코드블록, 표, 이미지 등 모듈별 활성화
- React 통합 패키지 (`@milkdown/react`) 공식 지원
- 오픈소스 (MIT 라이선스), 활발한 유지보수

**검증 필요 사항**: MVP 착수 전 프로토타입으로 한글 입력·표 WYSIWYG 편집 품질 확인 필요.

**대안 (TipTap)**:
- Milkdown 품질 미달 시 대안
- ProseMirror 기반, 더 큰 커뮤니티, 더 많은 레퍼런스
- 마크다운 완전 호환은 `@tiptap/extension-markdown` 플러그인 추가 필요

### 1.2 Zustand (탭 상태 관리)

- 탭 목록, 활성 탭, 각 탭의 에디터 상태를 Zustand 스토어로 관리
- 경량·단순 API로 복잡한 탭 상태를 예측 가능하게 관리

### 1.3 실제 구현 기술 스택 (Phase 1 기준)

| 기술 | 버전 | 비고 |
|------|------|------|
| Next.js | 16.1.6 | App Router (문서 표기 v15와 다름 — create-next-app@latest 결과) |
| React | 19.2.3 | |
| Tailwind CSS | v4 CSS-first | `tailwind.config.ts` 없음, `globals.css`의 `@theme inline` 블록 사용 |
| shadcn/ui | v4 + @base-ui/react | Radix UI 아님. `base-nova` 스타일 |
| Zustand | 5.0.11 | |
| @milkdown/kit + @milkdown/react + @milkdown/theme-nord | 7.19.0 | `nord` 테마는 `.config(nord)`로 적용 |
| next-themes | 최신 | 다크/라이트/시스템 테마 전환. `attribute="class"`, `defaultTheme="dark"` |

## 2. 기능 요구사항 (MoSCoW)

### Must Have

| 기능 ID | 기능 | 설명 |
|---------|------|------|
| ED-M1 | WYSIWYG 마크다운 편집 | Milkdown 기반 실시간 렌더링. 마크다운 문법 없이 서식 버튼으로 편집 가능 |
| ED-M2 | 마크다운 기본 요소 | 헤딩(H1-H6), **굵게**, *기울임*, ~~취소선~~, `인라인 코드`, 코드 블록, 링크, 이미지, 표, 인용, 순서/비순서 리스트, 수평선 |
| ED-M3 | .md 파일 열기 | 웹: File System Access API. 앱: Tauri FS API |
| ED-M4 | .md 파일 저장 | Ctrl+S. 웹: 다운로드 또는 File System Access API handle 저장. 앱: 원본 파일 덮어쓰기 |
| ED-M5 | 다른 이름으로 저장 | 파일명·저장 위치 선택하여 새 .md 파일 저장 |
| ED-M6 | **탭 기반 다중 문서** | 새 문서 생성·파일 열기 시 새 탭으로 처리. 상세 사항은 섹션 4 참조 |
| ED-M7 | 자동 저장 | 문서 변경 감지 후 30초 이내 자동 저장. 저장 시 탭에 상태 표시 |

## 4. 탭 기반 다중 문서 상세

### 4.1 탭 기본 동작

| 동작 | 설명 |
|------|------|
| 새 탭 생성 | "새 문서" 버튼 클릭 또는 Ctrl+T → 빈 에디터 탭 추가 |
| 파일 열기 → 탭 | 파일 열기 다이얼로그에서 파일 선택 → 새 탭으로 열림 |
| 중복 방지 | 이미 열린 파일 경로가 탭에 존재하면 새 탭 생성 없이 기존 탭으로 포커스 이동 |
| 탭 닫기 | 탭의 × 버튼 또는 Ctrl+W → 미저장 변경사항 있으면 확인 다이얼로그 표시 후 닫기 |
| 탭 전환 | 탭 클릭 또는 Ctrl+Tab (다음 탭), Ctrl+Shift+Tab (이전 탭) |
| 탭 재정렬 | 드래그 앤 드롭으로 탭 순서 변경 |

### 4.2 탭 상태 표시

| 상태 | 표시 방법 |
|------|----------|
| 파일명 | 탭에 파일명 표시. 새 문서는 "제목 없음" |
| 미저장 변경사항 | 탭 이름 앞에 `●` 표시 (예: `● 문서.md`) |
| 활성 탭 | 활성 탭 강조 스타일 (배경색, 하단 선) |
| 로딩 중 | 파일 읽기 중 스피너 표시 |

### 4.3 탭 상태 Zustand 스토어 구조

```typescript
interface TabStore {
  tabs: Tab[]              // 현재 열린 탭 목록
  activeTabId: string      // 활성 탭 ID
  openTab: (filePath?: string) => void    // 탭 열기 (중복 체크 포함)
  closeTab: (tabId: string) => void       // 탭 닫기 (미저장 체크)
  switchTab: (tabId: string) => void      // 탭 전환
  reorderTabs: (fromIdx: number, toIdx: number) => void  // 탭 재정렬
  updateTabContent: (tabId: string, content: string) => void  // 탭 내용 업데이트
  markTabDirty: (tabId: string, isDirty: boolean) => void     // 미저장 상태 표시
}

interface Tab {
  id: string               // 탭 고유 ID
  title: string            // 탭에 표시될 파일명
  filePath: string | null  // 로컬 파일 경로 (새 문서는 null)
  content: string          // 마크다운 내용
  isDirty: boolean         // 미저장 변경사항 존재 여부
  fileHandle?: FileSystemFileHandle  // 웹 File System Access API handle
}
```

## 5. 비기능 요구사항

### 5.1 성능

| 요구사항 | 목표 |
|---------|------|
| 타이핑 → 렌더링 지연 | P95 100ms 이하 |
| 파일 열기 시간 | 1MB 이하 .md 파일 1초 이내 |
| 탭 전환 시간 | 즉시 (100ms 이하) |
| 에디터 초기화 시간 | 첫 로딩 2초 이내 |

### 5.2 보안

| 요구사항 | 구현 |
|---------|------|
| 편집 데이터 서버 전송 없음 | Milkdown은 완전히 클라이언트 사이드에서 동작. 자동 저장도 로컬 파일에만 저장 |
| 파일 접근 최소 권한 | 웹: File System Access API (사용자가 명시적으로 허용한 파일만 접근). 앱: Tauri FS scope 설정으로 최소 권한 |
| XSS 방지 | Milkdown의 ProseMirror 기반 렌더링으로 사용자 입력 HTML 직접 삽입 없음 |

### 5.3 안정성

| 요구사항 | 목표 |
|---------|------|
| 파일 저장 성공률 | 99.9% 이상 |
| 에디터 오류율 | 세션당 0.1% 미만 |
| Undo 스택 유지 | 최소 100단계 |

## 6. UI/UX 요구사항

### 6.1 레이아웃

```
┌─────────────────────────────────────────────────────┐
│ [ModuMark 로고] [새 문서] [파일 열기]         [설정] │  ← 헤더
├──────┬──────┬──────┬─────────────────────────────── │
│ 탭1  │ 탭2● │ 탭3  │ +                              │  ← 탭 바
├──────┴──────┴──────┴────────────────────────────────┤
│ [서식 툴바: B I S ─ 링크 이미지 표 코드 ···]         │  ← 툴바
├─────────────────────────────────────────────────────┤
│                                                     │
│                  WYSIWYG 에디터 영역                 │
│                  (활성 탭 내용)                      │
│                                                     │
├─────────────────────────────────────────────────────┤
│ [WYSIWYG | Raw] 단어: 123 · 글자: 456 · [PDF 내보내기]│  ← 상태 바
└─────────────────────────────────────────────────────┘
```

### 6.2 핵심 UX 원칙

- **즉각 반응**: 버튼 클릭·키 입력에 100ms 이내 시각적 피드백
- **데이터 손실 방지**: 미저장 탭 닫기·앱 종료 시 반드시 확인
- **상태 명시**: 탭에 저장 상태, 상태 바에 문서 통계 항상 표시
- **접근성**: shadcn/ui v4 + @base-ui/react 기반으로 WCAG 2.1 AA 준수
