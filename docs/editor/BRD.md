## 1. 도메인 개요

### 1.1 도메인 목적

Editor 도메인은 ModuMark의 핵심 가치 제안인 **무료 마크다운 WYSIWYG 편집 경험**을 제공한다. Typora 유료화 이후 무료 대안을 찾는 사용자의 핵심 요구를 충족하는 도메인이다.

### 1.2 연결된 비즈니스 문제

| 전체 BRD 문제 ID | 문제 | Editor 도메인 연관성 |
|-----------------|------|-------------------|
| P1 | 마크다운 에디터와 PDF 편집 도구 분리 | 마크다운 편집 + PDF 변환 요청을 단일 인터페이스에서 제공 |
| P2 | 품질 있는 WYSIWYG 에디터가 유료 | **직접 해결**: 무료 WYSIWYG 편집 제공 |
| P4 | 데스크탑 전용 도구의 접근성 제한 | 웹에서도 동일한 편집 경험 제공 |

## 3. 이해관계자 (도메인 레벨)

| 이해관계자 | 관심사 | 요구사항 |
|-----------|--------|---------|
| 개발자 사용자 (페르소나: 김민준) | 빠른 마크다운 편집, GitHub README 작성 | 코드 블록 문법 강조, 빠른 키보드 단축키 |
| 학생 사용자 (페르소나: 이수진) | 수식·표 입력, PDF 제출 | 표 WYSIWYG 편집, 마크다운 모르는 사람도 사용 가능한 UI |
| 사무직 사용자 (페르소나: 박지현) | 마크다운 문법 몰라도 사용 가능 | 완전한 WYSIWYG (마크다운 문법 숨김) |
| 보안 민감 기업 사용자 (페르소나: 최성호) | 편집 중인 기밀 문서가 외부 서버로 전송되지 않아야 함 | 로컬 전용 편집 처리, 데이터 외부 전송 없음 보장 |

## 4. 핵심 비즈니스 규칙

| 규칙 ID | 규칙 | 근거 |
|---------|------|------|
| E-BR1 | 사용자 파일 및 편집 데이터는 어떠한 경우에도 외부 서버로 전송되거나 저장되지 않는다. 모든 편집 처리는 클라이언트(브라우저 또는 데스크탑 앱) 내에서만 수행된다 | 프라이버시 보호, 기업 보안 정책 준수, 로컬 우선 원칙 (BRD 8.1), 보안 페르소나(최성호) 요구사항 |
| E-BR2 | 문서 내용 변경 후 30초 이내 자동 저장이 실행된다 | 사용자 데이터 손실 방지 |
| E-BR3 | 저장되지 않은 변경사항이 있을 때 닫기 시 확인 다이얼로그를 표시한다 | 실수로 인한 데이터 손실 방지 |
| E-BR4 | 실행 취소(Undo) 스택은 최소 100단계를 유지한다 | 편집 안정성 확보 |
| E-BR5 | WYSIWYG 편집과 Raw 마크다운 편집 모드를 전환할 수 있다 | 파워 유저와 일반 사용자 모두 지원 |
| E-BR6 | Windows에서 .md 파일 더블클릭 시 ModuMark가 자동 실행된다 | 기존 Typora 사용자 워크플로우와 동일한 경험 |
| E-BR7 | 새 문서 생성 및 파일 열기는 새 창이 아닌 탭으로 열린다 | 여러 창이 분산되면 작업 컨텍스트가 단절되어 생산성이 저하됨 |
| E-BR8 | 탭은 동시에 여러 개를 유지할 수 있으며, 각 탭은 독립된 편집 상태를 가진다 | 탭 간 상태 충돌 방지 |
| E-BR9 | 저장되지 않은 탭을 닫으려 할 때 확인 다이얼로그를 표시한다 | 데이터 손실 방지 |
| E-BR10 | 이미 열린 파일을 다시 열기 시도하면 새 탭이 생성되지 않고 기존 탭으로 포커스가 이동한다 | 동일 파일 중복 편집 방지 |
| E-BR11 | 앱(Tauri) 환경에서 편집 중인 탭 내용을 5초 debounce로 백업 파일(`{APP_DATA_DIR}/backup/tab_{uuid}.md.bak`)에 자동 저장하고, 앱 재시작 시 session.json을 통해 탭을 복원한다 (Notepad++ 방식) | 앱 비정상 종료 또는 재시작 시 미저장 데이터 보호, PROPOSAL-005 채택 |
| E-BR12 | 웹 환경에서 IndexedDB 사용량이 50MB 소프트 한도를 초과하면 경고를 표시하고 데스크탑 앱 다운로드를 안내한다. 초과 후에도 작업은 차단하지 않는다 | 웹 스토리지 한계 명시, 앱 전환 유도, PROPOSAL-005 채택 |

## 6. 범위 정의

### 6.1 포함 범위 (In-Scope)

| 기능 | 설명 |
|------|------|
| WYSIWYG 편집 | 마크다운 렌더링 결과를 직접 편집 (Milkdown 기반) |
| 마크다운 요소 | 헤딩(H1-H6), 굵게, 기울임, 취소선, 코드(인라인/블록), 링크, 이미지, 표, 인용, 리스트(순서있음/없음), 수평선 |
| 파일 관리 | 새 문서, 파일 열기(.md), 저장, 다른 이름으로 저장 |
| **탭 기반 다중 문서** | 새 문서/파일 열기가 탭으로 처리됨. 탭 전환, 탭 닫기, 탭 재정렬 지원 |
| **탭 상태 표시** | 탭에 파일명 표시, 미저장 변경사항은 탭 이름에 `●` 또는 `*` 표시 |
| **중복 탭 방지** | 동일 파일 경로가 이미 탭에 열려 있으면 기존 탭으로 포커스 이동 |
| 자동 저장 | 변경 감지 후 30초 이내 자동 저장 |
| Undo/Redo | 100단계 편집 이력 |
| 편집 모드 전환 | WYSIWYG ↔ Raw Markdown |
| 단어/글자 수 표시 | 실시간 문서 통계 |
| 키보드 단축키 | Ctrl+B(굵게), Ctrl+I(기울임), Ctrl+S(저장), Ctrl+T(새 탭), Ctrl+W(탭 닫기) 등 |
| PDF 내보내기 트리거 | 편집 완료 후 PDF 변환 요청 버튼 |

### 6.2 제외 범위 (Out-of-Scope)

| 제외 항목 | 이유 |
|----------|------|
| 클라우드 동기화 | 로컬 우선 원칙, 백엔드 복잡도 |
| 실시간 협업 | 1인 개발 리소스 초과 |
| 플러그인 시스템 | Phase 1 MVP 범위 초과 |
| Mermaid 다이어그램 렌더링 | MVP 최소화 (Phase 2 검토) |
| 수식(LaTeX) 렌더링 | 초기 MVP 범위 초과 |
| 폴더/파일 트리 | MVP 범위 초과 |

## 7. 제약 조건

| 제약 | 내용 |
|------|------|
