## 1. 도메인 개요

### 1.1 도메인 목적

Platform 도메인은 **Windows 네이티브 앱과 웹 브라우저라는 두 가지 실행 환경을 추상화**하여 Editor/PDF 도메인이 플랫폼에 독립적으로 동작할 수 있게 한다. SEO 최적화된 웹 서비스 기반과 데스크탑 앱 배포를 담당한다.

### 1.2 연결된 비즈니스 문제

| 전체 BRD 문제 ID | 문제 | Platform 도메인 연관성 |
|-----------------|------|----------------------|
| P4 | 데스크탑 전용 도구의 접근성 제한 | **직접 해결**: 웹 + 데스크탑 동시 지원 |
| P1 | 여러 도구 간 컨텍스트 전환 | .md 파일 기본 앱 등록으로 단일 앱 워크플로우 구현 |

## 3. 이해관계자 (도메인 레벨)

| 이해관계자 | 관심사 | 요구사항 |
|-----------|--------|---------|
| 웹 사용자 (이수진) | 설치 없이 즉시 사용 | 빠른 초기 로딩, 모바일 반응형 |
| 데스크탑 사용자 (김민준) | .md 파일 연결, 오프라인 사용 | 파일 연결 등록, 오프라인 동작 |
| 보안 민감 기업 사용자 (최성호) | 로컬 우선 아키텍처로 기업 보안 정책 준수 | 파일 전송 없는 서버리스/클라이언트 처리 구조, 데스크탑 앱 오프라인 동작 |
| Google (검색 엔진) | 크롤링 가능한 구조 | SSR/SSG, 시맨틱 HTML, sitemap |
| Google AdSense | 트래픽 품질, 콘텐츠 충분성 | 충분한 텍스트 콘텐츠 페이지 |

## 4. 핵심 비즈니스 규칙

| 규칙 ID | 규칙 | 근거 |
|---------|------|------|
| PL-BR1 | 웹 공개 페이지는 SSR 또는 SSG로 렌더링된다 | SEO 검색 엔진 크롤링 필수, AdSense 승인 조건 |
| PL-BR2 | 모든 공개 페이지는 고유한 title과 meta description을 가진다 | SEO 최적화, CTR 개선 |
| PL-BR3 | sitemap.xml과 robots.txt를 자동 생성하여 제공한다 | 검색 엔진 인덱싱 최적화 |
| PL-BR4 | Windows 배포 파일은 코드 서명(Code Signing)을 적용한다 | SmartScreen 경고 방지, 사용자 신뢰 확보 |
| PL-BR5 | .md 파일 기본 앱으로 ModuMark를 등록할 수 있다 | 기존 Typora 사용자 워크플로우 동일 경험 |
| PL-BR6 | 앱 업데이트는 사용자 동의 후 설치된다 | 사용자 통제권 보장 |
| PL-BR7 | 웹과 데스크탑에서 동일한 핵심 기능을 제공한다 | 크로스 플랫폼 일관된 경험 |
| PL-BR8 | 모든 파일 처리(편집, PDF 변환, OCR 등)는 클라이언트에서 수행되며, 파일 데이터가 외부 서버로 전송되지 않는다 | 로컬 우선 아키텍처, 기업 보안 정책 준수, 개인정보보호법 (보안 페르소나: 최성호) |
| PL-BR9 | "보안 안전한 로컬 우선" 아키텍처를 제품 소개 페이지에 명시적으로 안내한다 | 보안 민감 사용자 대상 차별점 커뮤니케이션 |

## 6. 범위 정의

### 6.1 포함 범위 (In-Scope)

| 기능 | 설명 |
|------|------|
| Next.js 웹 앱 | SSR/SSG 기반 Next.js 16.1.6 App Router 웹 서비스 |
| SEO 기반 구조 | 메타태그, OG 태그, sitemap.xml, robots.txt, 구조화 데이터 |
| Tauri 데스크탑 앱 | Windows 네이티브 앱 (Tauri 2.0 + Next.js) |
| 파일 시스템 접근 | 로컬 .md/.pdf 파일 읽기/쓰기 (Tauri FS API) |
| .md 파일 연결 | Windows에서 .md 파일을 ModuMark로 여는 기본 앱 등록 |
| 자동 업데이터 | 새 버전 감지 및 사용자 동의 후 업데이트 설치 |
| 창 상태 관리 | 창 크기·위치 저장/복원, 최대화/최소화 |
| 앱 설정 | 테마(라이트/다크/시스템), 언어(한국어) 설정 |
| 반응형 레이아웃 | 모바일(375px) ~ 데스크탑(1440px) 대응 |
| Vercel 배포 | 웹 앱 서버리스 배포 ✅ Phase 1 완료 |
| **테마 전환** | 다크/라이트/시스템 테마 전환 (next-themes) ✅ Phase 1 완료 |

### 6.2 제외 범위 (Out-of-Scope)

| 제외 항목 | 이유 |
|----------|------|
| macOS / Linux 앱 | 1차 출시 범위 초과 |
| Microsoft Store 배포 | 심사 프로세스 복잡, 직접 배포 우선 |
| PWA (Progressive Web App) | Phase 2 이후 검토 |
| 다국어 UI (영어 등) | 한국 시장 집중 후 확장 |
| 오프라인 웹 모드 | 서비스 워커 복잡도, Phase 3 검토 |
